I often think of today's weather relative to the weather yesterday or in the past few days. This visualization compares today's weather data to yesterday's data and sizes the bars accordingly. For example, one can view if the temperature today is warmer or colder than yesterday by seeing if the first bar is positive or negative. The scaling of the bars are adjustable to users' preferences. The weather icon at the top indicates the general weather of today, if the user just wants a quick summary. The interface is hard to figure out on its own due to its simplicity, but if this were an app, first-time users would be directed to a description page before using it.

This app would be for users who only care about the weather when they are not busy with indoor activities, such as attending class. The columns represent the days from Sunday to Saturday, and the rows represent hours within the day. The hours can be customized to each person's preferences. For example, I am most active 11am to 12am, so the top row represents 11am-12pm and the bottom row represents 11pm-12am. For each hour of the week, if the user isn't busy, a dot is displayed. The dot's color represents the temperature range for that hour. As with the first dashboard, the interface is hard to figure out on its own due to its simplicity, but if this were an app, first-time users would be directed to a description page before using it.
